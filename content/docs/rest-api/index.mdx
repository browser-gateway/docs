The gateway exposes three categories of HTTP endpoints:

- **Action endpoints** (`/v1/screenshot`, `/v1/content`, `/v1/scrape`): one-shot browser actions, no client code needed.
- **Control endpoints** (`/v1/providers`, `/v1/config`, `/v1/profiles`, `/v1/replays`, `/v1/sessions`, `/v1/status`): manage the gateway itself.
- **Compat + dashboard** (`/health`, `/json/version`, `/web/*`): Chrome-compat endpoints and the dashboard.

All `/v1/*` endpoints require the `BG_TOKEN` header (or `?token=` query param) if `BG_TOKEN` is set. `/health` and `/json/version` are always public.

## By resource

- [**Auth Info**](/rest-api/auth): Report whether the gateway requires an auth token.
- [**Config**](/rest-api/config): Read, validate, and update gateway.yml over HTTP.
- [**Content**](/rest-api/content): Extract page content (HTML, markdown, or text) from any URL.
- [**Health**](/rest-api/health): Liveness probe. Always public, no auth required.
- [**Chrome Compat**](/rest-api/json-version): Chrome-compatible /json/version endpoint.
- [**Profiles**](/rest-api/profiles): Create, export, import, delete stored browser profiles.
- [**Providers**](/rest-api/providers): CRUD for provider entries in gateway.yml.
- [**Replays**](/rest-api/replays): List, fetch, delete session replay frames.
- [**Scrape**](/rest-api/scrape): Structured data extraction from a page.
- [**Screenshot**](/rest-api/screenshot): One-shot screenshot of any URL, no client code required.
- [**Sessions**](/rest-api/sessions): Read-only view of active WebSocket sessions.
- [**Status**](/rest-api/status): Aggregate gateway status (providers, sessions, pool).
- [**Dashboard**](/rest-api/web): Dashboard static files + login endpoints.
